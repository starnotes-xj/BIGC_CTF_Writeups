## 解题过程

### 1. 初始侦察/文件识别
- 附件给出一组 RSA 参数：

```text
N = ...
e = 65537
c = ...
```

- 题目提示：
  - `N` 的值很特殊
  - 目标是把 `N` 分解成两个质数
  - 关键在于 `1` 和 `7` 的个数相同
  - 示例（按乘法结构理解）：`11111 × 1000007 = 11111077777`

- 观察附件中的 `N`，它的十进制形式非常异常，只由大量 `1`、`0`、`7` 组成：

```text
111...111000...000777...777
```

### 2. 关键突破点一
- 统计 `N` 的十进制数字分布：

```text
len(N) = 729
count('1') = 317
count('0') = 95
count('7') = 317
```

- 因此 `N` 的结构可以写成：

```text
N = 1...1 0...0 7...7
    317   95    317
```

- 令：

\[
R_n = \frac{10^n - 1}{9}
\]

也就是由 `n` 个 `1` 组成的 repunit。

- 题目给的例子：

```text
11111 × 1000007 = 11111077777
```

本质上就是：

\[
R_5 \cdot (10^6+7)
= R_5\cdot 10^6 + 7R_5
= 11111077777
\]

- 一般化后，如果：

```text
N = 1^a 0^b 7^a
```

那么可以尝试：

\[
p = R_a = \frac{10^a-1}{9}
\]

\[
q = 10^{a+b}+7
\]

并检查：

\[
N = p q
\]

### 3. 关键突破点二
- 对本题：

```text
a = 317
b = 95
```

- 所以：

\[
p = \frac{10^{317}-1}{9}
\]

\[
q = 10^{412}+7
\]

- 直接验证：

```python
p = (10**317 - 1) // 9
q = 10**412 + 7
assert p * q == N
```

- 这两个数也正好都是质数，因此 RSA 模数成功分解。

### 4. 获取 Flag
- 得到 `p, q` 后就是标准 RSA 解密：

\[
\varphi(N)=(p-1)(q-1)
\]

\[
d=e^{-1}\bmod\varphi(N)
\]

\[
m=c^d\bmod N
\]

- 用 Python 还原明文字节：

```python
phi = (p - 1) * (q - 1)
d = pow(e, -1, phi)
m = pow(c, d, N)
flag = m.to_bytes((m.bit_length() + 7) // 8, "big")
```

- 得到：

```text
CPCTF{N_1s_34sy_70_bRe4k_873b4982a}
```

## 攻击链/解题流程总结

```text
读取 N/e/c -> 观察 N 的十进制异常结构 -> 统计 1/0/7 个数 -> 利用 1^a0^b7^a = repunit(a) * (10^(a+b)+7) -> 分解 N -> 标准 RSA 解密 -> 得到 Flag
```

## 漏洞分析 / 机制分析

### 根因
- RSA 安全性依赖于模数 `N = p*q` 难以分解。
- 本题的 `N` 不是随机生成的普通 RSA 模数，而是被构造成了非常明显的十进制模式。
- 当 `N` 呈现 `1^a0^b7^a` 这种形式，并且 `1` 与 `7` 的个数相同，就能反推出两个因子：

\[
\frac{10^a-1}{9}
\quad\text{和}\quad
10^{a+b}+7
\]

### 影响
- 不需要通用大整数分解算法。
- 只要识别十进制模式并套用题目提示，就能瞬间得到 `p` 和 `q`。
- 后续解密完全退化为普通 RSA 私钥恢复。

### 修复建议（适用于漏洞类题目）
- 真实 RSA 模数必须由安全随机生成的独立大素数相乘得到。
- 不应使用有明显十进制结构、可预测结构或人为构造模式的素数 / 模数。
- 生成密钥时应使用成熟密码库，不要自行构造“看起来有趣”的 `p`、`q` 或 `N`。

## 知识点

- Repunit 数 \(R_n=(10^n-1)/9\)
- 从十进制模式反推乘法结构
- 题目提示中的小例子如何一般化

## 推荐工具与优化解题流程

> 这题的核心不是跑通用分解，而是识别 `N` 的十进制结构异常。

### 工具对比总结

| 工具 | 适用阶段 | 本题耗时 | 优点 | 缺点 |
|------|----------|----------|------|------|
| 直接观察十进制 `N` | 建模 | 秒级 | 能发现大量 `1/0/7` 的模式 | 需要注意数字个数 |
| Python 脚本 | 解密 | 秒级 | 适合统计、构造因子、计算 RSA 私钥 | 需要写少量代码 |
| 通用因数分解 | 误区 / 备选 | 不定 | 不依赖模式识别 | 本题没有必要，可能很慢 |

### 推荐流程

**推荐流程**：先观察 `N` 的十进制结构 -> 统计连续 `1/0/7` 的个数 -> 构造 `p=(10^a-1)//9` 和 `q=10^(a+b)+7` -> 验证 `p*q==N` -> RSA 解密得到 flag。

### 工具 A（推荐首选）
- **安装**：Python 3
- **详细步骤**：
  1. 正则读取附件中的 `N, e, c`
  2. 对 `str(N)` 做 run-length 统计
  3. 得到 `a = count('1')`、`b = count('0')`
  4. 构造 `p` 和 `q`
  5. 用 `pow(e, -1, phi)` 求私钥指数并解密
- **优势**：完全贴合题目提示，最快最稳

### 工具 B（可选）
- **安装**：SageMath / SymPy
- **详细步骤**：可用 `is_prime` / `isprime` 验证 `p, q` 是否为质数。
- **优势**：适合补充验证，但不是解题必要条件
